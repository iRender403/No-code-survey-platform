# 里程碑

## 低代码平台的底层逻辑

- 低码世界
在低代码的视角下，页面的本质是一串结构化的数据，而非静态的代码。用户的UI操作，实质上是在用JavaScript修改这串数据
```json
  {
    "id":???,
    "componet":"Button",
    "parentId":"canvas_root",
    "styles":{
        "position":"",
        "left":"",
        "top":""
    },
  }

```
那么数据是如何变成画面的？这里涉及两个细节
- 渲染
1. 组件映射
浏览器无法直接渲染字符串。我们需要通过一个"组件映射表，将数据中标识符关联到真实的代码上
2. 树形映射
商业级的页面并非的简单的堆砌，而是复杂的嵌套关系。因此我们采用树结构来描述逻辑的准确性
